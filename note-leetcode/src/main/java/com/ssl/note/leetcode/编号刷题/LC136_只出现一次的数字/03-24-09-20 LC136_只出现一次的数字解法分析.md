# LC136_只出现一次的数字
## 题目描述
给定一个非空整数数组，除了某个元素只出现一次以外，其余每个元素均出现两次。找出那个只出现了一次的元素。
说明：你的算法应该具有线性时间复杂度。 你可以不使用额外空间来实现吗？
示例1：
```
输入：[2,2,1]
输出：1
```
示例2：
```
输入：[4,1,2,1,2]
输出：4
```
示例3：
```
输入：[1]
输出：1
```
## 解法概览
| 解法 | 思路 | 核心公式 | 时间复杂度 | 空间复杂度 |
|------|------|----------|------------|------------|
| 异或运算 | 利用异或性质 | `a ^ a = 0`, `a ^ 0 = a` | O(n) | O(1) |
| 哈希表 | 统计频率 | 键值对存储 | O(n) | O(n) |
| 集合 | 数学公式 | 2*(a+b+c) - (a+a+b+b+c) = c | O(n) | O(n) |
## 记忆口诀
只出现一次，异或运算；两两抵消，剩下唯一
## 解法一：异或运算（最优）
### 思路
利用异或运算的性质：
- 任何数与自身异或结果为 0（`a ^ a = 0`）
- 任何数与 0 异或结果为自身（`a ^ 0 = a`）
- 异或运算满足交换律和结合律
遍历数组，将所有元素异或，出现两次的元素会互相抵消，最终结果就是只出现一次的元素。
### 核心公式
- 初始值：`single = 0`
- 循环：`single ^= num`
- 最终：`single` 就是只出现一次的数
### 图解过程
```
nums = [4,1,2,1,2]

single = 0
single ^ 4 → 4
4 ^ 1 → 5
5 ^ 2 → 7
7 ^ 1 → 6
6 ^ 2 → 4

最终：4
```
### 代码示例
```java
public int singleNumber(int[] nums) {
    int single = 0;
    for (int num : nums) {
        // 异或运算：相同为0，不同为1
        // 出现两次的数会互相抵消，剩下只出现一次的数
        single ^= num;
    }
    return single;
}
```
### 复杂度分析
- 时间复杂度：O(n)，只遍历数组一次
- 空间复杂度：O(1)，无需额外空间
### 优缺点
- 优点：时间复杂度最优，空间复杂度为常数
- 缺点：只适用于其他元素出现偶数次的情况
### 测试用例
- 示例1：[2,2,1] → 1
- 示例2：[4,1,2,1,2] → 4
- 示例3：[1] → 1
- 边界：数组长度为1 → 直接返回
### 边界情况分析
- 数组长度为1：直接返回该元素
- 数组长度为奇数：正常异或
- 所有元素都相同：不可能，题目保证有且只有一个元素出现一次
### 其他解法
#### 哈希表解法
```java
public int singleNumber(int[] nums) {
    Map<Integer, Integer> map = new HashMap<>();
    for (int num : nums) {
        map.put(num, map.getOrDefault(num, 0) + 1);
    }
    for (Map.Entry<Integer, Integer> entry : map.entrySet()) {
        if (entry.getValue() == 1) {
            return entry.getKey();
        }
    }
    return -1;
}
```
- 时间复杂度：O(n)
- 空间复杂度：O(n)

#### 集合解法
```java
public int singleNumber(int[] nums) {
    Set<Integer> set = new HashSet<>();
    int sum = 0;
    int uniqueSum = 0;
    for (int num : nums) {
        if (!set.contains(num)) {
            set.add(num);
            uniqueSum += num;
        }
        sum += num;
    }
    return 2 * uniqueSum - sum;
}
```
- 时间复杂度：O(n)
- 空间复杂度：O(n)
### 解题技巧总结
- 技巧1：异或运算的性质是解决此类问题的关键
- 技巧2：交换律和结合律保证了顺序不影响结果
- 技巧3：常数空间复杂度是本题的最优解
