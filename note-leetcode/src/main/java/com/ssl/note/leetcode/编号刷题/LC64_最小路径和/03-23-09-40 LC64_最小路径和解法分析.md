# LC64_最小路径和
## 题目描述
给定一个包含非负整数的 m x n 网格 grid ，请找出一条从左上角到右下角的路径，使得路径上的数字总和为最小。
说明：每次只能向下或者向右移动一步。
示例1：
```
输入：grid = [[1,3,1],[1,5,1],[4,2,1]]
输出：7
解释：路径 1→3→1→1→1 的和最小
```
示例2：
```
输入：grid = [[1,2,3],[4,5,6]]
输出：12
解释：路径 1→2→3→6 的和最小
```
## 解法概览
| 解法 | 思路 | 核心公式 | 时间复杂度 | 空间复杂度 |
|------|------|----------|------------|------------|
| 二维DP | 自左上角到右下角动态规划 | `dp[i][j]=min(dp[i-1][j],dp[i][j-1])+grid[i][j]` | O(m*n) | O(m*n) |
| 一维DP | 空间优化 | `dp[j]=min(dp[j],dp[j-1])+grid[i][j]` | O(m*n) | O(n) |
## 记忆口诀
最小路径和，DP来求；只能右走或下走，取小加上当前
## 解法一：二维动态规划
### 思路
创建一个二维 dp 数组，dp[i][j] 表示从左上角到达位置 (i,j) 的最小路径和。由于只能从上方或左方到达当前格子，因此取两者较小者加上当前格子的值。
### 核心公式
- `dp[0][0] = grid[0][0]` 起点
- `dp[0][j] = dp[0][j-1] + grid[0][j]` 第一行，只能从左边来
- `dp[i][0] = dp[i-1][0] + grid[i][0]` 第一列，只能从上边来
- `dp[i][j] = min(dp[i-1][j], dp[i][j-1]) + grid[i][j]` 其他位置，取小加上当前
### 图解过程
```
grid = [[1,3,1],[1,5,1],[4,2,1]]

dp表填充过程：
     j=0   j=1   j=2
i=0  [1]   [4]   [5]    ← 1→3→1=5
i=1  [2]   [7]   [6]    ← min(4,2)+5=7, min(7,5)+1=6
i=2  [6]   [8]   [7]    ← min(2,6)+4=6, min(7,8)+2=8, min(6,8)+1=7

答案：dp[2][2] = 7
```
### 代码示例
```java
public int minPathSum(int[][] grid) {
    if (grid == null || grid.length == 0) {
        return 0;
    }
    int m = grid.length;
    int n = grid[0].length;
    int[][] dp = new int[m][n];
    dp[0][0] = grid[0][0];

    // 初始化第一行：只能从左边过来
    for (int i = 1; i < n; i++) {
        dp[0][i] = dp[0][i - 1] + grid[0][i];
    }

    // 初始化第一列：只能从上边过来
    for (int i = 1; i < m; i++) {
        dp[i][0] = dp[i - 1][0] + grid[i][0];
    }

    // 填表：每个位置取 min(上方, 左边) + 当前值
    for (int i = 1; i < m; i++) {
        for (int j = 1; j < n; j++) {
            dp[i][j] = Math.min(dp[i - 1][j], dp[i][j - 1]) + grid[i][j];
        }
    }

    return dp[m - 1][n - 1];
}
```
### 复杂度分析
- 时间复杂度：O(m*n)，遍历整个网格
- 空间复杂度：O(m*n)，存储 dp 表
### 优缺点
- 优点：思路清晰，容易理解
- 缺点：空间可以优化
### 测试用例
- 示例1：grid=[[1,3,1],[1,5,1],[4,2,1]] → 期望输出：7
- 示例2：grid=[[1,2,3],[4,5,6]] → 期望输出：12
- 边界：m=1 或 n=1 → 只走一条直线
### 边界情况分析
- 网格为空：返回 0
- 单行：只能向右走，路径唯一
- 单列：只能向下走，路径唯一
- 包含 0：0 不影响最小路径计算
### 易错点提醒
- 易错点1：忘记初始化第一行和第一列
- 易错点2：dp[0][0] 要赋值为 grid[0][0] 而非 0
- 易错点3：取 min 时不要遗漏当前格子的值
### 优化版本
```java
// 一维数组空间优化
public int minPathSum(int[][] grid) {
    int m = grid.length;
    int n = grid[0].length;
    int[] dp = new int[n];

    for (int i = 0; i < m; i++) {
        for (int j = 0; j < n; j++) {
            if (i == 0 && j == 0) {
                dp[j] = grid[i][j];
            } else if (i == 0) {
                dp[j] = dp[j - 1] + grid[i][j];
            } else if (j == 0) {
                dp[j] = dp[j] + grid[i][j];
            } else {
                dp[j] = Math.min(dp[j], dp[j - 1]) + grid[i][j];
            }
        }
    }
    return dp[n - 1];
}
```
### 解题技巧总结
- 技巧1：机器人只能右走或下走，所以状态只依赖上方和左边
- 技巧2：初始化第一行和第一列时只有一种选择
- 技巧3：空间优化时，dp[j] 旧值是上方，dp[j-1] 是左边
