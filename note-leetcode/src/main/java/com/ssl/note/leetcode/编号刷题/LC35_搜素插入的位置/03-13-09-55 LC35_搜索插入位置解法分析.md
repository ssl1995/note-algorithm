# 03-13-09-55 LC35_搜索插入位置解法分析
## 题目描述
给定一个排序数组和一个目标值，在数组中找到目标值，并返回其索引。如果目标值不存在于数组中，返回它将会被按顺序插入的位置。请必须使用时间复杂度为 O(log n) 的算法。
**示例：**
输入：nums = [1,3,5,6], target = 5
输出：2
输入：nums = [1,3,5,6], target = 2
输出：1
输入：nums = [1,3,5,6], target = 7
输出：4
输入：nums = [1,3,5,6], target = 0
输出：0

## 记忆口诀
**二分查找：** 有序数组找位置，左闭右开要牢记；等于返回索引值，插入返回左边界。
## 不同解法
### 解法一：二分查找（最优解）
#### 思路
利用数组有序的特性，使用二分查找快速定位目标值或插入位置。定义左右指针，通过比较中间元素与目标值的大小来不断缩小搜索区间，最终找到目标值的插入位置。
#### 核心公式
- 循环条件：left < right
- 中间索引：mid = left + (right - left) / 2
- 移动条件：如果 target <= nums[mid]，则 right = mid；否则 left = mid + 1
- 返回值：循环结束后的 left 即为插入位置
#### 图解过程
以 nums = [1,3,5,6], target = 2 为例：
1. 初始：left=0, right=4, mid=2, nums[2]=5 > 2, right=2
2. left=0, right=2, mid=1, nums[1]=3 > 2, right=1
3. left=0, right=1, mid=0, nums[0]=1 < 2, left=1
4. left=1, right=1, 循环结束，返回 left=1
#### 代码示例
```java
public class Solution {
    public int searchInsert(int[] nums, int target) {
        int left = 0;
        int right = nums.length;
        while (left < right) {
            int mid = left + (right - left) / 2;
            if (target <= nums[mid]) {
                right = mid;
            } else {
                left = mid + 1;
            }
        }
        return left;
    }
}
```
#### 复杂度分析
- 时间复杂度：O(log n)，每次循环将搜索区间缩小一半
- 空间复杂度：O(1)，只使用了常数个变量
#### 优缺点
- 优点：时间复杂度最优，符合题目要求
- 缺点：需要理解二分查找的边界处理
### 解法二：暴力遍历（普通解法）
#### 思路
从数组开头依次遍历，找到第一个大于等于目标值的位置，或者遍历到数组末尾。
#### 核心公式
- 遍历条件：for (int i = 0; i < nums.length; i++)
- 返回条件：if (nums[i] >= target) return i
- 循环结束：return nums.length
#### 图解过程
以 nums = [1,3,5,6], target = 2 为例：
1. i=0: nums[0]=1 < 2，继续
2. i=1: nums[1]=3 >= 2，返回 1
#### 代码示例
```java
public class Solution {
    public int searchInsert(int[] nums, int target) {
        for (int i = 0; i < nums.length; i++) {
            if (nums[i] >= target) {
                return i;
            }
        }
        return nums.length;
    }
}
```
#### 复杂度分析
- 时间复杂度：O(n)，最坏情况需要遍历整个数组
- 空间复杂度：O(1)，只使用了常数个变量
#### 优缺点
- 优点：代码简单，容易理解
- 缺点：时间复杂度为 O(n)，不满足题目要求
## 面试回答模板
**问题：** 请实现搜索插入位置算法。
**回答：**
这是一道经典的二分查找题目。给定一个排序数组，需要找到目标值的索引，或者返回它应该插入的位置。
我主要使用二分查找解法，时间复杂度为 O(log n)，符合题目要求。
具体思路是：维护左右指针，每次取中间元素与目标值比较，如果目标值小于等于中间元素，说明插入位置在左半部分，将右指针移动到中间位置；否则将左指针移动到中间位置的右侧。循环直到左指针与右指针相遇，此时左指针的位置就是目标值的插入位置。
二分查找的关键点在于边界处理：我使用左闭右开的区间 [left, right)，循环条件是 left < right，这样当循环结束时，left 就是目标值的插入位置。
**示例：** 对于 nums = [1,3,5,6], target = 2，经过二分查找，最终返回 1，表示 2 应该插入到索引 1 的位置。
