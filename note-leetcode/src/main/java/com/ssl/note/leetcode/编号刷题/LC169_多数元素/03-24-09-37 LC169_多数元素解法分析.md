# LC169_多数元素
## 题目描述
给定一个大小为 n 的数组 nums ，返回其中的多数元素。多数元素是指在数组中出现次数 大于 ⌊ n/2 ⌋ 的元素。
你可以假设数组是非空的，并且给定的数组总是存在多数元素。
示例1：
```
输入：nums = [3,2,3]
输出：3
```
示例2：
```
输入：nums = [2,2,1,1,1,2,2]
输出：2
```
## 解法概览
| 解法 | 思路 | 核心公式 | 时间复杂度 | 空间复杂度 |
|------|------|----------|------------|------------|
| 摩尔投票 | 候选者与抵消 | 相同+1，不同-1，为0换候选 | O(n) | O(1) |
| 哈希表 | 统计频率 | 键值对存储 | O(n) | O(n) |
| 排序 | 取中间元素 | 排序后 nums[n/2] | O(n log n) | O(1) |
## 记忆口诀

## 解法一：摩尔投票算法（最优）
### 思路
摩尔投票算法的核心思想是：
- 维护一个候选元素和一个计数器
- 遍历数组，遇到相同元素计数器加1，遇到不同元素计数器减1
- 当计数器为0时，更换候选元素并重置计数器为1
- 最终的候选元素一定是多数元素
### 核心公式
- 初始化：`candidate = nums[0], count = 1`
- 遍历：
  - `nums[i] == candidate` → `count++`
  - `nums[i] != candidate` → `count--`
  - `count == 0` → `candidate = nums[i], count = 1`
- 最终：`candidate` 即为多数元素
### 图解过程
```
nums = [2,2,1,1,1,2,2]

过程：
candidate=2, count=1 → 遇到2 → count=2
candidate=2, count=2 → 遇到1 → count=1
candidate=2, count=1 → 遇到1 → count=0
count=0 → candidate=1, count=1
candidate=1, count=1 → 遇到1 → count=2
candidate=1, count=2 → 遇到2 → count=1
candidate=1, count=1 → 遇到2 → count=0
count=0 → candidate=2, count=1

最终：candidate=2
```
### 代码示例
```java
public int majorityElement(int[] nums) {
    int candidate = nums[0];
    int count = 1;

    for (int i = 1; i < nums.length; i++) {
        if (candidate == nums[i]) {
            count++;
        } else {
            count--;
            if (count == 0) {
                candidate = nums[i];
                count = 1;
            }
        }
    }

    return candidate;
}
```
### 复杂度分析
- 时间复杂度：O(n)，只遍历数组一次
- 空间复杂度：O(1)，无需额外空间
### 优缺点
- 优点：时间复杂度最优，空间复杂度为常数
- 缺点：只适用于存在多数元素的情况（题目保证存在）
### 测试用例
- 示例1：[3,2,3] → 3
- 示例2：[2,2,1,1,1,2,2] → 2
- 边界：[1] → 1
- 边界：[1,1,1,2,2] → 1
### 边界情况分析
- 数组长度为1：直接返回该元素
- 多数元素在开头：正常处理
- 多数元素在中间：正常处理
- 多数元素在结尾：正常处理
### 其他解法
#### 哈希表解法
```java
public int majorityElement(int[] nums) {
    Map<Integer, Integer> map = new HashMap<>();
    for (int num : nums) {
        map.put(num, map.getOrDefault(num, 0) + 1);
        if (map.get(num) > nums.length / 2) {
            return num;
        }
    }
    return -1;
}
```
- 时间复杂度：O(n)
- 空间复杂度：O(n)

#### 排序解法
```java
public int majorityElement(int[] nums) {
    Arrays.sort(nums);
    return nums[nums.length / 2];
}
```
- 时间复杂度：O(n log n)
- 空间复杂度：O(1)（原地排序）
### 解题技巧总结
- 技巧1：摩尔投票算法是解决多数元素问题的最优解
- 技巧2：利用多数元素出现次数超过一半的特性
- 技巧3：计数器的增减模拟了投票过程
