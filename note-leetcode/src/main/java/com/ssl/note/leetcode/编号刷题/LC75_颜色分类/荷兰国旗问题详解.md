# 荷兰国旗问题详解
## 一、问题起源
荷兰国旗问题（Dutch National Flag Problem）是由计算机科学家 Edsger W. Dijkstra 提出的经典算法问题。该问题的名称来源于荷兰国旗的颜色组成：红、白、蓝三色，且这三种颜色在国旗中是按顺序排列的。
**问题描述**：
给定一个包含三种颜色元素的数组，要求将它们按照特定的顺序排列，使得相同颜色的元素相邻。在 LeetCode 75 题中，这三种颜色分别用 0（红）、1（白）、2（蓝）表示，要求排列为 0、1、2 的顺序。
## 二、核心思路
荷兰国旗问题的核心思路是**三区间划分**，使用三个指针将数组分为三个区间：
1. **[0, left)**：0 区（红色）
2. **[left, cur)**：1 区（白色）
3. **[cur, right]**：未处理区
4. **(right, n-1]**：2 区（蓝色）
通过遍历未处理区的元素，根据其值调整指针位置，最终将所有元素分类到对应的区间。
## 三、算法步骤
1. **初始化指针**：
   - `left = 0`：0 区的右边界
   - `cur = 0`：当前处理的位置
   - `right = nums.length - 1`：2 区的左边界
2. **遍历未处理区**：
   - 当 `cur <= right` 时，循环处理：
     - **情况1**：`nums[cur] == 0`
       - 将当前元素交换到 0 区
       - `left++`，`cur++`（0 区扩大，继续处理下一个元素）
     - **情况2**：`nums[cur] == 1`
       - 元素属于 1 区，直接 `cur++`（1 区扩大）
     - **情况3**：`nums[cur] == 2`
       - 将当前元素交换到 2 区
       - `right--`（2 区扩大，`cur` 不变，因为交换过来的元素还需要处理）
3. **结束条件**：
   - 当 `cur > right` 时，所有元素处理完毕
## 四、代码实现
```java
public void sortColors(int[] nums) {
    int left = 0;        // 0区的右边界
    int cur = 0;         // 当前处理的位置
    int right = nums.length - 1;  // 2区的左边界

    while (cur <= right) {
        if (nums[cur] == 0) {
            // 交换到0区
            swap(nums, left, cur);
            left++;
            cur++;
        } else if (nums[cur] == 1) {
            // 属于1区，直接右移
            cur++;
        } else {
            // 交换到2区
            swap(nums, cur, right);
            right--;
            // cur不变，因为交换过来的元素还需要处理
        }
    }
}

private void swap(int[] nums, int i, int j) {
    int temp = nums[i];
    nums[i] = nums[j];
    nums[j] = temp;
}
```
## 五、示例解析
### 示例：`nums = [2,0,2,1,1,0]`
| 步骤 | left | cur | right | nums | 操作 |
|------|------|-----|-------|------|------|
| 初始 | 0    | 0   | 5     | [2,0,2,1,1,0] | - |
| 1    | 0    | 0   | 4     | [0,0,2,1,1,2] | 交换 nums[0] 和 nums[5] |
| 2    | 1    | 1   | 4     | [0,0,2,1,1,2] | nums[0]==0，left++，cur++ |
| 3    | 2    | 2   | 4     | [0,0,2,1,1,2] | nums[1]==0，left++，cur++ |
| 4    | 2    | 2   | 3     | [0,0,1,1,2,2] | 交换 nums[2] 和 nums[4] |
| 5    | 2    | 3   | 3     | [0,0,1,1,2,2] | nums[2]==1，cur++ |
| 6    | 2    | 4   | 3     | [0,0,1,1,2,2] | cur > right，结束 |
**结果**：`[0,0,1,1,2,2]`
## 六、复杂度分析
- **时间复杂度**：O(n)，只需一次遍历数组，每个元素最多被处理一次
- **空间复杂度**：O(1)，原地排序，无需额外空间
## 七、应用场景
1. **颜色分类**：如 LeetCode 75 题，将三种颜色按顺序排列
2. **三值排序**：处理只有三种可能值的数组排序
3. **分区问题**：将数组按照某个条件分为三个部分
4. **快速排序的分区步骤**：快速排序中也使用了类似的分区思想
## 八、与其他排序算法的对比
| 算法 | 时间复杂度 | 空间复杂度 | 特点 |
|------|------------|------------|------|
| 荷兰国旗算法 | O(n) | O(1) | 一次遍历，原地排序，适用于三值问题 |
| 计数排序 | O(n) | O(1) | 两次遍历，思路简单 |

## 九、扩展与变体
### 1. 四值问题
如果需要处理四种值的排序，可以扩展为四区间划分，使用四个指针。
### 2. 通用化
对于 k 种值的排序，荷兰国旗算法的思想可以扩展为 k 区间划分，但时间复杂度会相应增加。
### 3. 稳定性
荷兰国旗算法是不稳定的排序算法，因为交换操作可能会改变相同元素的相对顺序。
## 十、面试常见问题
### 1. 荷兰国旗算法的时间复杂度是多少？
**回答**：O(n)，只需要一次遍历数组，每个元素最多被处理一次。
### 2. 为什么在处理 2 时 `cur` 不需要右移？
**回答**：因为从右边交换过来的元素可能是 0、1 或 2，还需要进一步处理，所以 `cur` 保持不变，继续检查交换过来的元素。
### 3. 如何处理更多颜色的情况？
**回答**：对于 k 种颜色，可以扩展为 k 区间划分，使用 k+1 个指针，但时间复杂度会增加到 O(kn)。
### 4. 荷兰国旗算法与快速排序的关系？
**回答**：快速排序中的分区步骤使用了类似的思想，将数组分为小于、等于和大于 pivot 的三个部分。
## 十一、总结
荷兰国旗问题是一个经典的排序问题，其核心思想是通过三指针法实现三区间划分，能够在 O(n) 时间复杂度和 O(1) 空间复杂度的情况下完成排序。这种算法不仅解决了颜色分类问题，也为其他类似的分区问题提供了思路。
**关键要点**：
- 三指针法的区间划分思想
- 不同情况下的指针移动策略
- 一次遍历完成排序的高效性
- 原地排序的空间优势
通过掌握荷兰国旗算法，我们可以更深刻地理解分区排序的思想，为解决类似问题提供有力的工具。
