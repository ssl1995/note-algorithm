# LC31_下一个排列
## 题目描述
实现获取 下一个排列 的函数，算法需要将给定数字序列重新排列成字典序中下一个更大的排列。
如果不存在下一个更大的排列，则将数字重新排列成最小的排列（即升序排列）。
必须 原地 修改，只允许使用额外常数空间。
示例1：
```
输入：nums = [1,2,3]
输出：[1,3,2]
```
示例2：
```
输入：nums = [3,2,1]
输出：[1,2,3]
```
示例3：
```
输入：nums = [1,1,5]
输出：[1,5,1]
```
示例4：
```
输入：nums = [1]
输出：[1]
```
## 解法概览

|------|------|----------|------------|------------|
| 标准算法 | 三步骤：找下降点、交换、反转 | 无特定公式 | O(n) | O(1) |

## 记忆口诀
下一个排列，三步要牢记：找下降点，交换大值，反转剩余
## 解法一：标准算法（最优解）
### 思路
1. **找下降点**：从右往左遍历，找到第一个满足 `nums[i] < nums[i+1]` 的位置 i，这是需要调整的起始点
2. **交换大值**：从右往左遍历，找到第一个满足 `nums[j] > nums[i]` 的位置 j，交换 nums[i] 和 nums[j]
3. **反转剩余**：反转 i+1 到末尾的所有元素，得到下一个排列
### 核心步骤
1. 寻找下降点 i：`while (i >= 0 && nums[i] >= nums[i+1]) { i--; }`
2. 寻找交换点 j：`while (j >= i && nums[j] <= nums[i]) { j--; }`
3. 交换元素：`swap(nums, i, j)`
4. 反转剩余部分：`reverseAfterI(nums, i+1, nums.length-1)`
### 图解过程
以 nums = [4,5,2,6,3,1] 为例：
1. **找下降点**：从右往左遍历，i=2（nums[2]=2 < nums[3]=6）
2. **找交换点**：从右往左遍历，找到第一个比2大的数，j=4（nums[4]=3）
3. **交换**：交换 nums[2] 和 nums[4]，得到 [4,5,3,6,2,1]
4. **反转**：反转 i+1=3 到末尾的元素，得到 [4,5,3,1,2,6]
### 代码示例
```java
public void nextPermutation(int[] nums) {
    if (nums.length < 2) {
        return;
    }
    // 1.从倒数第二个数往前跳过找，找到i位置使得nums[i]<nums[i+1]
    int i = nums.length - 2;
    while (i >= 0 && nums[i] >= nums[i + 1]) {
        i--;
    }
    // 2.如果i>=0存在，从倒数第一个数开始往前，找第一个比它大的数交换
    if (i >= 0) {
        int j = nums.length - 1;
        while (j >= i && nums[j] <= nums[i]) {
            j--;
        }
        swap(nums, i, j);
    }
    // 3.逆置[i+1,n-1]的数
    reverseAfterI(nums, i + 1, nums.length - 1);
}

private void swap(int[] nums, int i, int j) {
    if (i == j) {
        return;
    }
    int temp = nums[i];
    nums[i] = nums[j];
    nums[j] = temp;
}

private void reverseAfterI(int[] nums, int start, int end) {
    while (start < end) {
        swap(nums, start, end);
        start++;
        end--;
    }
}
```
### 复杂度分析
- 时间复杂度：O(n)，最多需要三次遍历数组（找下降点、找交换点、反转）
- 空间复杂度：O(1)，原地修改，只使用常数级额外空间
### 优缺点
- 优点：时间复杂度最优，空间复杂度为常数，实现简单高效
- 缺点：逻辑需要仔细理解，特别是边界情况的处理
### 测试用例
- 示例1：[1,2,3] → 期望输出：[1,3,2]
- 示例2：[3,2,1] → 期望输出：[1,2,3]
- 示例3：[1,1,5] → 期望输出：[1,5,1]
- 示例4：[1] → 期望输出：[1]
- 边界测试：[1,3,2] → 期望输出：[2,1,3]
- 边界测试：[2,3,1] → 期望输出：[3,1,2]
### 边界情况分析
- 数组长度小于2：直接返回，无需处理
- 数组已为最大排列（如[3,2,1]）：i=-1，直接反转整个数组得到最小排列
- 数组元素全部相同（如[2,2,2]）：i=-1，直接反转，结果不变
### 易错点提醒
- 找下降点时，条件应为 `nums[i] >= nums[i+1]`，注意包含等于的情况
- 找交换点时，条件应为 `nums[j] <= nums[i]`，同样包含等于的情况
- 当i=-1时（无下降点），需要反转整个数组
- 交换后必须反转i+1到末尾的部分，否则无法得到正确的下一个排列
### 解题技巧总结
- 技巧1：从右往左遍历寻找下降点，比从左往右更高效
- 技巧2：交换后反转剩余部分，利用已有的有序性
- 技巧3：处理边界情况，确保算法的鲁棒性
