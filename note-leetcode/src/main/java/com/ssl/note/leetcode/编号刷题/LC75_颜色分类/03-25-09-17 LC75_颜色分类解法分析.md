# LC75_颜色分类
## 题目描述
给定一个包含红色、白色和蓝色，一共 n 个元素的数组，原地对它们进行排序，使得相同颜色的元素相邻，并按照红色、白色、蓝色的顺序排列。
此题中，我们使用整数 0、1 和 2 分别表示红色、白色和蓝色。
示例1：
```
输入：nums = [2,0,2,1,1,0]
输出：[0,0,1,1,2,2]
```
示例2：
```
输入：nums = [2,0,1]
输出：[0,1,2]
```
## 解法概览
| 解法 | 思路 | 核心公式 | 时间复杂度 | 空间复杂度 |
|------|------|----------|------------|------------|
| 计数排序 | 统计0、1、2的出现次数 | `count[num]++` | O(n) | O(1) |
| 双指针 | 三区间划分 | 0区[0,left)，1区[left,cur)，未处理[cur,right]，2区(right,n-1] | O(n) | O(1) |
| 单指针 | 两次遍历 | 先放0，再放1 | O(n) | O(1) |

## 解法一：计数排序
### 思路
统计数组中0、1、2的出现次数，然后按照0、1、2的顺序重新填充数组。
### 核心公式
- 统计：`count[num]++`
- 填充：按顺序写入0、1、2
### 代码示例
```java
public void sortColors(int[] nums) {
    // 计数数组，统计0、1、2的出现次数
    int[] count = new int[3];
    for (int num : nums) {
        count[num]++;
    }

    // 按顺序填充0、1、2
    for (int i = 0; i < count[0]; i++) {
        nums[i] = 0;
    }
    for (int i = count[0]; i < count[0] + count[1]; i++) {
        nums[i] = 1;
    }
    for (int i = count[0] + count[1]; i < nums.length; i++) {
        nums[i] = 2;
    }
}
```
### 复杂度分析
- 时间复杂度：O(n)，两次遍历数组
- 空间复杂度：O(1)，使用固定大小的计数数组
### 优缺点
- 优点：思路简单，实现容易
- 缺点：需要两次遍历数组
## 解法二：双指针（荷兰国旗问题）
### 思路
使用三个指针将数组分为三个区间：
- [0, left)：0 区
- [left, cur)：1 区
- [cur, right]：未处理区
- (right, n-1]：2 区
遍历过程中，根据当前元素的值调整指针位置。
### 核心公式
- `nums[cur] == 0`：交换到0区，left和cur都右移
- `nums[cur] == 1`：属于1区，cur右移
- `nums[cur] == 2`：交换到2区，right左移，cur不变
### 图解过程
```
nums = [2,0,2,1,1,0]

初始：left=0, cur=0, right=5

cur=0: nums[0]=2 → 交换nums[0]和nums[5] → [0,0,2,1,1,2], right=4, cur=0
cur=0: nums[0]=0 → 属于0区，left=1, cur=1
cur=1: nums[1]=0 → 属于0区，left=2, cur=2
cur=2: nums[2]=2 → 交换nums[2]和nums[4] → [0,0,1,1,2,2], right=3, cur=2
cur=2: nums[2]=1 → 属于1区，cur=3
cur=3: nums[3]=1 → 属于1区，cur=4
cur=4: 超过right=3，结束
```
### 代码示例
```java
public void sortColors(int[] nums) {
    int left = 0;        // 0区的右边界
    int cur = 0;         // 当前处理的位置
    int right = nums.length - 1;  // 2区的左边界

    while (cur <= right) {
        if (nums[cur] == 0) {
            // 交换到0区
            swap(nums, left, cur);
            left++;
            cur++;
        } else if (nums[cur] == 1) {
            // 属于1区，直接右移
            cur++;
        } else {
            // 交换到2区
            swap(nums, cur, right);
            right--;
            // cur不变，因为交换过来的元素还需要处理
        }
    }
}

private void swap(int[] nums, int i, int j) {
    int temp = nums[i];
    nums[i] = nums[j];
    nums[j] = temp;
}
```
### 复杂度分析
- 时间复杂度：O(n)，一次遍历完成
- 空间复杂度：O(1)，原地排序
### 优缺点
- 优点：一次遍历，原地排序，效率高
- 缺点：逻辑稍复杂，需要仔细处理指针移动
### 测试用例
- 示例1：[2,0,2,1,1,0] → [0,0,1,1,2,2]
- 示例2：[2,0,1] → [0,1,2]
- 边界：[0,0,0] → [0,0,0]
- 边界：[1,1,1] → [1,1,1]
- 边界：[2,2,2] → [2,2,2]
### 边界情况分析
- 数组全为0：left和cur同步右移
- 数组全为1：cur一直右移
- 数组全为2：right一直左移
- 元素已排序：一次遍历完成
### 其他解法
#### 单指针解法
```java
public void sortColors(int[] nums) {
    int ptr = 0;
    // 先放0
    for (int i = 0; i < nums.length; i++) {
        if (nums[i] == 0) {
            swap(nums, ptr, i);
            ptr++;
        }
    }
    // 再放1
    for (int i = ptr; i < nums.length; i++) {
        if (nums[i] == 1) {
            swap(nums, ptr, i);
            ptr++;
        }
    }
    // 剩下的都是2
}
```
- 时间复杂度：O(n)
- 空间复杂度：O(1)
### 解题技巧总结
- 技巧1：计数排序适用于元素范围小的情况
- 技巧2：双指针法（荷兰国旗问题）是本题的最优解
- 技巧3：三区间划分的思想可以推广到其他类似问题
