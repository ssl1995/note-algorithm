# 排列、组合、子集概念详解
## 一、用生活例子秒懂三个概念
### 排列（Permutation）— 有顺序
```
3个人（A、B、C）排队站成一排，有几种站法？
  ABC、ACB、BAC、BCA、CAB、CBA → 6种
AB 和 BA 是不同的排列（A在前 vs B在前）
关键词：顺序不同 = 不同的结果
```
### 组合（Combination）— 没有顺序
```
3个人（A、B、C）选2个人去开会，有几种选法？
  AB、AC、BC → 3种
AB 和 BA 是同一种选法（都是A和B去开会）
关键词：只关心"选了谁"，不关心"谁先谁后"
```
### 子集（Subset）— 选或不选
```
3个人（A、B、C），列出所有可能的小组（包括空组和全组）：
  {}、{A}、{B}、{C}、{AB}、{AC}、{BC}、{ABC} → 8种
关键词：每个人要么选要么不选，2^n 种
```

## 二、核心区别一览表
| 概念 | 关心顺序？ | 选几个？ | 结果数 | 举例（从ABC中选2个） |
|------|----------|---------|--------|-------------------|
| **排列** | ✅ 关心 | 固定个数 | n!/(n-k)! | AB,BA,AC,CA,BC,CB=6 |
| **组合** | ❌ 不关心 | 固定个数 | C(n,k) | AB,AC,BC=3 |
| **子集** | ❌ 不关心 | 所有可能 | 2^n | {},A,B,C,AB,AC,BC,ABC=8 |
| **全排列** | ✅ 关心 | **全部选** | n! | ABC,ACB,BAC,BCA,CAB,CBA=6 |
> **全排列 = 排列的特殊情况，所有元素都要选上，只是顺序不同。**

## 三、数学公式
### 排列数 P(n,k) — 从n个中选k个，有顺序
```
P(n,k) = n! / (n-k)!
P(3,2) = 3! / 1! = 6 / 1 = 6
全排列：P(n,n) = n! / 0! = n!
  P(3,3) = 3! = 6
```
### 组合数 C(n,k) — 从n个中选k个，无顺序
```
C(n,k) = n! / (k! × (n-k)!)
C(3,2) = 3! / (2! × 1!) = 6 / 2 = 3
组合 = 排列 / k!（去掉顺序的重复）
```
### 子集数 — 每个元素选或不选
```
2^n
2^3 = 8（3个元素的所有子集）
```

## 四、用代码区分三者
三者用的都是**回溯模板**，区别只有两点：**for循环起点**和**收集时机**。
### 统一框架
```java
void backtrack(int[] nums, int start/i, List<Integer> path, ...) {
    收集结果的时机;
    for (int i = 起点; i < nums.length; i++) {
        path.add(nums[i]);     // 选择
        backtrack(...);         // 递归
        path.removeLast();      // 撤销
    }
}
```
### 排列（LC46）— 每次从头选 + used标记
```java
// 关键：for从0开始，用used防止重复选
void backtrack(int[] nums, List<Integer> path, boolean[] used) {
    if (path.size() == nums.length) { res.add(new ArrayList<>(path)); return; }
    for (int i = 0; i < nums.length; i++) {     // ← 从0开始！
        if (used[i]) continue;                   // ← 用过的跳过
        path.add(nums[i]); used[i] = true;
        backtrack(nums, path, used);
        path.remove(path.size()-1); used[i] = false;
    }
}
```
### 组合（LC77）— 从start开始，不回头
```java
// 关键：for从start开始，保证不选前面的（去重）
void backtrack(int[] nums, int start, List<Integer> path) {
    if (path.size() == k) { res.add(new ArrayList<>(path)); return; }
    for (int i = start; i < nums.length; i++) {  // ← 从start开始！
        path.add(nums[i]);
        backtrack(nums, i + 1, path);             // ← 下一层从i+1开始
        path.remove(path.size()-1);
    }
}
```
### 子集（LC78）— 从start开始，每个节点都收集
```java
// 关键：每个节点都是一个子集，不只是叶子
void backtrack(int[] nums, int start, List<Integer> path) {
    res.add(new ArrayList<>(path));               // ← 每个节点都收集！
    for (int i = start; i < nums.length; i++) {   // ← 从start开始
        path.add(nums[i]);
        backtrack(nums, i + 1, path);
        path.remove(path.size()-1);
    }
}
```
### 三者代码差异总结
| 差异点 | 排列 | 组合 | 子集 |
|-------|------|------|------|
| for起点 | `i = 0`（每次从头） | `i = start`（不回头） | `i = start`（不回头） |
| 防重复 | `used[i]` 标记 | 天然不重复（只往后选）| 天然不重复 |
| 收集时机 | `path.size == n` | `path.size == k` | **每个节点都收集** |
| 递归参数 | `backtrack(used)` | `backtrack(i+1)` | `backtrack(i+1)` |

## 五、决策树对比（nums=[1,2,3]）
### 排列的决策树（每层都能选所有未用过的）
```
                   []
          /         |         \
        [1]        [2]        [3]
       /   \      /   \      /   \
    [1,2] [1,3] [2,1] [2,3] [3,1] [3,2]
      |     |     |     |     |     |
  [1,2,3] ...   ...   ...   ...  [3,2,1]
叶子节点收集：6个排列
```
### 组合的决策树（k=2，只往后选）
```
                    []
          /          |         \
        [1]         [2]        [3]
       /   \         |
    [1,2] [1,3]    [2,3]
叶子节点收集：3个组合
注意：没有[2,1]，因为只能往后选
```
### 子集的决策树（每个节点都收集）
```
                     []  ← 收集
          /           |          \
        [1] ← 收集  [2] ← 收集  [3] ← 收集
       /   \          |
    [1,2]  [1,3]    [2,3]
  ← 收集  ← 收集   ← 收集
      |
   [1,2,3] ← 收集
所有节点都收集：8个子集
```

## 六、面试常见的相似概念区分
### 1. 排列 vs 组合
```
排列：[1,2] 和 [2,1] 是不同的  → 班长和副班长（谁当班长有区别）
组合：[1,2] 和 [2,1] 是相同的  → 选两个人去开会（不分先后）
代码区别：排列for从0+used，组合for从start
```
### 2. 子集 vs 组合
```
子集：选0个、1个、2个...n个的所有组合 = 所有大小的组合之和
组合：只选固定k个
C(n,0) + C(n,1) + ... + C(n,n) = 2^n = 子集总数
所以：子集 = 所有k的组合的并集
```
### 3. 全排列 vs 排列
```
全排列：n个元素全部选上，只是顺序不同 → P(n,n) = n!
排列：n个元素选k个，考虑顺序 → P(n,k)
全排列是排列的特例（k=n）
```
### 4. 有重复 vs 无重复
```
无重复（LC46/77/78）：
  元素都不同，直接回溯
有重复（LC47/40/90）：
  元素可能重复，需要额外剪枝
  排序 + "同层跳过相同元素"
  if (i > start && nums[i] == nums[i-1]) continue;
```
### 5. 可重复选 vs 不可重复选
```
不可重复选（LC46/77/78）：
  每个元素只能用一次
  递归传 i+1
可重复选（LC39 组合总和）：
  每个元素可以用多次
  递归传 i（不是i+1，可以再选自己）
```

## 八、一句话总结
```
排列：关心顺序，[1,2]≠[2,1] → for从0+used
组合：不关心顺序，[1,2]=[2,1] → for从start
子集：所有大小的组合         → for从start+每个节点都收集
全排列：排列的特例，全部选上  → LC46
记住这张表，所有排列组合子集题都是同一个回溯模板的变体
```
