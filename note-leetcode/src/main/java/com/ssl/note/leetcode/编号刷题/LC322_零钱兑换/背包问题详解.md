# 背包问题详解
## 1. 0-1 背包问题
### 1.1 基本概念
0-1 背包问题（0-1 Knapsack Problem）是最经典的背包问题：
- **物品使用次数**：每种物品只能选择 **0 个或 1 个**
- **核心限制**：每个物品要么完全放入背包，要么完全不放入
### 1.2 问题描述
给定：
- `n` 种物品，每种物品的重量为 `w[i]`，价值为 `v[i]`
- 一个容量为 `C` 的背包
目标：
选择若干物品放入背包，使得总重量不超过 `C`，且总价值最大。
### 1.3 解法
#### 状态定义
- `dp[j]`：背包容量为 `j` 时的最大价值
#### 状态转移方程
- `dp[j] = max(dp[j], dp[j-w[i]] + v[i])`
#### 遍历顺序
- **逆序遍历容量**：`j = C downto w[i]`
- **原因**：确保每个物品只被选择一次
#### 代码实现
```java
public int knapsack01(int[] w, int[] v, int C) {
    int n = w.length;
    int[] dp = new int[C+1];
    for (int i = 0; i < n; i++) {
        for (int j = C; j >= w[i]; j--) {  // 逆序遍历
            dp[j] = Math.max(dp[j], dp[j - w[i]] + v[i]);
        }
    }
    return dp[C];
}
```
## 2. 完全背包问题
### 2.1 基本概念
完全背包问题（Complete Knapsack Problem）是背包问题的一种变体：
- **物品使用次数**：每种物品可以选择 **无限次**
- **核心特点**：物品可以重复使用
### 2.2 问题描述
给定：
- `n` 种物品，每种物品的重量为 `w[i]`，价值为 `v[i]`
- 一个容量为 `C` 的背包
目标：
选择若干物品放入背包，使得总重量不超过 `C`，且总价值最大。
### 2.3 解法
#### 状态定义
- `dp[j]`：背包容量为 `j` 时的最大价值
#### 状态转移方程
- `dp[j] = max(dp[j], dp[j-w[i]] + v[i])`
#### 遍历顺序
- **正序遍历容量**：`j = w[i] to C`
- **原因**：允许物品重复选择
#### 代码实现
```java
public int completeKnapsack(int[] w, int[] v, int C) {
    int n = w.length;
    int[] dp = new int[C+1];
    for (int i = 0; i < n; i++) {
        for (int j = w[i]; j <= C; j++) {  // 正序遍历
            dp[j] = Math.max(dp[j], dp[j - w[i]] + v[i]);
        }
    }
    return dp[C];
}
```
### 2.4 与 0-1 背包的区别
| 特性 | 0-1 背包 | 完全背包 |
|------|----------|----------|
| 物品使用次数 | 0 或 1 | 无限次 |
| 遍历顺序 | 逆序遍历容量 | 正序遍历容量 |
| `dp[j-w[i]]` 含义 | 未考虑当前物品 | 已考虑当前物品的多次使用 |
## 3. 零钱兑换问题
### 3.1 问题描述
给定不同面额的硬币 `coins` 和一个总金额 `amount`，计算凑成总金额所需的最少硬币个数。如果没有任何一种硬币组合能组成总金额，返回 -1。
### 3.2 问题分析
- **问题类型**：完全背包的变体（硬币可重复使用）
- **目标**：最小化硬币个数
- **约束**：总金额等于 `amount`
### 3.3 解法
#### 状态定义
- `dp[i]`：凑成金额 `i` 所需的最少硬币个数
#### 状态转移方程
- `dp[i] = min(dp[i], dp[i-coin] + 1)`
#### 初始化
- `dp[0] = 0`（凑成金额 0 不需要硬币）
- 其他 `dp[i] = amount + 1`（初始化为无穷大，表示不可达）
#### 代码实现
```java
public int coinChange(int[] coins, int amount) {
    int[] dp = new int[amount + 1];
    // 初始化为无穷大
    for (int i = 1; i <= amount; i++) {
        dp[i] = amount + 1;
    }
    dp[0] = 0;
    for (int coin : coins) {
        for (int i = coin; i <= amount; i++) {
            dp[i] = Math.min(dp[i], dp[i - coin] + 1);
        }
    }
    return dp[amount] > amount ? -1 : dp[amount];
}
```
### 3.4 关键要点
- **完全背包特性**：硬币可以重复使用，所以正序遍历金额
- **最小值问题**：使用 `min` 操作，初始化为无穷大
- **不可达判断**：如果 `dp[amount]` 仍为无穷大，返回 -1
## 4. 三种问题的对比
| 特性 | 0-1 背包 | 完全背包 | 零钱兑换 |
|------|----------|----------|----------|
| 物品类型 | 不可重复 | 可重复 | 可重复 |
| 目标函数 | 最大值 | 最大值 | 最小值 |
| 状态转移 | max | max | min |
| 初始化 | 0 | 0 | 0（其他为无穷大） |
| 遍历顺序 | 逆序 | 正序 | 正序 |
| 核心方程 | `max(dp[j], dp[j-w[i]]+v[i])` | `max(dp[j], dp[j-w[i]]+v[i])` | `min(dp[i], dp[i-coin]+1)` |

### 5.3 零钱兑换示例

```java
public class CoinChange {

        int[] dp = new int[amount + 1];

            dp[i] = amount + 1;
        }

        for (int coin : coins) {

                dp[i] = Math.min(dp[i], dp[i - coin] + 1);
            }
        }
        return dp[amount] > amount ? -1 : dp[amount];
    }
}
```
## 6. 总结
### 6.1 核心要点
- **0-1 背包**：物品不可重复，逆序遍历容量
- **完全背包**：物品可重复，正序遍历容量
- **零钱兑换**：完全背包的变体，求最小值，初始化为无穷大
### 6.2 解题技巧
1. **问题识别**：判断物品是否可重复使用
2. **状态定义**：根据问题目标定义合适的状态
3. **转移方程**：根据问题类型（最大值、最小值、计数）确定转移方程
4. **初始化**：根据问题类型设置合适的初始值
5. **遍历顺序**：0-1背包逆序，完全背包正序
### 6.3 应用价值
背包问题是动态规划中的经典问题，其思想和解法被广泛应用于：
- 资源分配问题
- 组合优化问题
- 金融投资问题
- 生产计划问题
- 货币兑换问题
通过掌握这三种背包问题的解法，我们可以解决很多实际生活中的优化问题，体现了算法在解决实际问题中的重要价值。
