# 拓扑排序建图详解
## 一、用穿衣服来理解拓扑排序
```
你早上穿衣服有先后顺序：
  先穿内裤 → 才能穿外裤
  先穿袜子 → 才能穿鞋
  先穿衬衫 → 才能穿西装
这些"先后关系"就是有向边：
  内裤 → 外裤
  袜子 → 鞋
  衬衫 → 西装
拓扑排序 = 找出一个合法的穿衣顺序，保证所有先后关系都满足
比如：内裤→袜子→衬衫→外裤→鞋→西装 ✅
```
## 二、建图过程分三步
### 总体流程
```
输入：一堆"A必须在B前面"的关系
   ↓
第1步：建邻接表（谁指向谁）
第2步：算入度（每个节点有几个前置）
第3步：BFS（从没有前置的开始，一层层推进）
```
### 以 LC207 课程表为例
```
输入：numCourses = 4
      prerequisites = [[1,0], [2,0], [3,1], [3,2]]
含义：
  [1,0] → 学1之前要先学0 → 0指向1
  [2,0] → 学2之前要先学0 → 0指向2
  [3,1] → 学3之前要先学1 → 1指向3
  [3,2] → 学3之前要先学2 → 2指向3
```

## 三、第1步：建邻接表
### 邻接表是什么？
```
邻接表 = 记录"每个节点指向谁"的表
就像通讯录：每个人后面跟着他能联系到的人
  0 → [1, 2]     0学完之后可以学1和2
  1 → [3]         1学完之后可以学3
  2 → [3]         2学完之后可以学3
  3 → []          3学完之后没有后续了
```
### 怎么读题目的输入？
```
prerequisites[i] = [ai, bi] 表示"学ai前要先学bi"
  [ai, bi] → bi 是前置，ai 是后续
  方向：bi → ai（从前置指向后续）
⚠️ 注意：不是 ai→bi！很多人在这里搞反
记忆方法：
  [1, 0] → "1需要0" → 0是前置 → 0→1
  谁是前置，谁在箭头前面
  或者：先修课(bi) → 后续课(ai)
```
### 建邻接表的代码
```java
// 初始化：每个节点都有一个空列表
List<List<Integer>> graph = new ArrayList<>();
for (int i = 0; i < numCourses; i++) {
    graph.add(new ArrayList<>());
}
// 读入边：bi → ai
for (int[] pre : prerequisites) {
    int ai = pre[0];  // 后续课
    int bi = pre[1];  // 前置课（先修）
    graph.get(bi).add(ai);  // bi 指向 ai
}
```
### 逐条建图过程
```
初始：graph = [[], [], [], []]
              课程0 课程1 课程2 课程3
处理 [1,0]：0→1
  graph[0].add(1) → graph = [[1], [], [], []]
处理 [2,0]：0→2
  graph[0].add(2) → graph = [[1,2], [], [], []]
处理 [3,1]：1→3
  graph[1].add(3) → graph = [[1,2], [3], [], []]
处理 [3,2]：2→3
  graph[2].add(3) → graph = [[1,2], [3], [3], []]
最终邻接表：
  0 → [1, 2]
  1 → [3]
  2 → [3]
  3 → []
对应的图：
  0 → 1 → 3
  0 → 2 ↗
```

## 四、第2步：算入度
### 入度是什么？
```
入度 = 有几条箭头指向这个节点 = 有几门先修课
  0 → 1 → 3
  0 → 2 ↗
  节点0：没人指向它 → 入度0（可以直接开始）
  节点1：0指向它   → 入度1
  节点2：0指向它   → 入度1
  节点3：1和2指向它 → 入度2
生活类比：
  入度 = 你被几个人拦着 = 你前面有几道门
  入度0 = 前面没人拦你 = 可以直接出发
```
### 算入度的代码
```java
int[] inDegree = new int[numCourses]; // 初始全是0
for (int[] pre : prerequisites) {
    int ai = pre[0];  // ai 是被指向的（后续课）
    inDegree[ai]++;    // ai 多了一个前置
}
```
### 逐条计算过程
```
初始：inDegree = [0, 0, 0, 0]
处理 [1,0]：课程1多了一个前置 → inDegree[1]++
  inDegree = [0, 1, 0, 0]
处理 [2,0]：课程2多了一个前置 → inDegree[2]++
  inDegree = [0, 1, 1, 0]
处理 [3,1]：课程3多了一个前置 → inDegree[3]++
  inDegree = [0, 1, 1, 1]
处理 [3,2]：课程3又多了一个前置 → inDegree[3]++
  inDegree = [0, 1, 1, 2]
含义：
  课程0：0门先修 → 可以直接学
  课程1：1门先修（0）
  课程2：1门先修（0）
  课程3：2门先修（1和2）→ 最后才能学
```

## 五、第3步：BFS拓扑排序
### 过程
```
第1步和第2步已经完成了"地图"的绘制：
  邻接表（谁指向谁）+ 入度（有几个前置）
现在开始BFS：
  1. 把入度0的节点入队（没有前置，可以开始）
  2. 出队一个节点 = 完成这门课
  3. 它的后续课程入度减1 = 先修要求少了一个
  4. 如果某门课入度变成0 = 所有先修都完成了 → 入队
  5. 重复直到队列为空
```
### 完整过程图解
```
邻接表：0→[1,2]  1→[3]  2→[3]  3→[]
入度：  [0, 1, 1, 2]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
入度0的入队：课程0
queue = [0]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
出队0，学完课程0
  0的后续：1和2
  inDegree[1]: 1→0 → 入队！
  inDegree[2]: 1→0 → 入队！
  queue = [1, 2]
  学习顺序：[0]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
出队1，学完课程1
  1的后续：3
  inDegree[3]: 2→1 → 还有前置没完成
  queue = [2]
  学习顺序：[0, 1]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
出队2，学完课程2
  2的后续：3
  inDegree[3]: 1→0 → 所有前置完成！入队！
  queue = [3]
  学习顺序：[0, 1, 2]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
出队3，学完课程3
  3的后续：无
  queue = []

4门课全部出队 = 可以完成 ✅
```

## 六、建图的两种实现方式
### 方式一：List<List<Integer>>（推荐）
```java
List<List<Integer>> graph = new ArrayList<>();
for (int i = 0; i < n; i++) {
    graph.add(new ArrayList<>());
}
graph.get(from).add(to);
// 访问：graph.get(node) 得到邻居列表
```
### 方式二：Map<Integer, List<Integer>>
```java
Map<Integer, List<Integer>> graph = new HashMap<>();
for (int i = 0; i < n; i++) {
    graph.put(i, new ArrayList<>());
}
graph.get(from).add(to);
// 访问：graph.get(node) 得到邻居列表
```
### 对比
| 方式 | 适用场景 | 优点 | 缺点 |
|------|---------|------|------|
| List<List> | 节点编号是0~n-1 | 更快，下标直接访问 | 节点必须是连续整数 |
| Map<Integer, List> | 节点编号不连续 | 通用 | 稍慢（哈希查找） |
> LC207 节点是 0~n-1，用 List<List> 更好。

## 七、常见易错点
### 错误1：边的方向搞反
```
[1, 0] 表示"学1前要先学0"
  正确：0 → 1（0指向1）
  错误：1 → 0（方向反了）
记忆：[后续, 前置]，前置指向后续
  pre[1] → pre[0]
  graph.get(pre[1]).add(pre[0])
```
### 错误2：入度算反了
```
[1, 0] → 课程1被依赖了（被0指向）
  正确：inDegree[1]++（1的入度+1）
  错误：inDegree[0]++（0是前置，入度应该不变）
记忆：谁是被指向的（后续课），谁的入度++
  inDegree[pre[0]]++
```
### 错误3：建图和算入度可以合并
```java
// 建图和入度一起处理，一次遍历搞定
for (int[] pre : prerequisites) {
    graph.get(pre[1]).add(pre[0]);  // 建图
    inDegree[pre[0]]++;             // 算入度
}
```

## 八、拓扑排序建图模板总结
```java
// ====== 模板 ======
// 1. 初始化邻接表
List<List<Integer>> graph = new ArrayList<>();
for (int i = 0; i < n; i++) graph.add(new ArrayList<>());
// 2. 建图 + 算入度
int[] inDegree = new int[n];
for (int[] edge : edges) {
    int from = edge[1];  // 前置（注意题目的方向）
    int to = edge[0];    // 后续
    graph.get(from).add(to);
    inDegree[to]++;
}
// 3. 入度0的入队
Queue<Integer> queue = new LinkedList<>();
for (int i = 0; i < n; i++) {
    if (inDegree[i] == 0) queue.offer(i);
}
// 4. BFS
int count = 0;
while (!queue.isEmpty()) {
    int node = queue.poll();
    count++;
    for (int next : graph.get(node)) {
        if (--inDegree[next] == 0) {
            queue.offer(next);
        }
    }
}
// 5. 判断
return count == n;  // 全部出队 = 无环
```
## 九、一句话总结
```
建图就3件事：
  1. 邻接表：记录"谁的后面跟着谁"（graph[from].add(to)）
  2. 入度：记录"谁前面有几个人"（inDegree[to]++）
  3. 注意方向：[后续, 前置] → 前置指向后续
然后BFS：入度0先走，走完给后续减1，减到0就走
```
