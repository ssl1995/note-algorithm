# 摩尔投票算法详解
## 一、算法简介
摩尔投票算法（Moore Voting Algorithm）是一种用于解决**多数元素问题**的高效算法。该算法由 Robert S. Boyer 和 J Strother Moore 于 1981 年提出，用于在 O(n) 时间复杂度和 O(1) 空间复杂度的情况下找出数组中的多数元素。
**多数元素**：在数组中出现次数超过数组长度一半的元素。
## 二、核心思想
摩尔投票算法的核心思想是**投票抵消**：
1. **候选者**：维护一个候选元素
2. **计数器**：维护一个计数器，记录候选元素的「支持度」
3. **投票规则**：
   - 遇到与候选元素相同的元素 → 计数器 +1
   - 遇到与候选元素不同的元素 → 计数器 -1
   - 当计数器为 0 时 → 更换候选元素，重置计数器为 1
## 三、原理推导
### 1. 正确性证明
**假设**：数组中存在多数元素  m ，出现次数  > n/2 。
**过程**：
- 当遇到  m  时，计数器 +1（支持）
- 当遇到非  m  时，计数器 -1（反对）
- 由于  m  的出现次数超过一半，最终计数器必定为正数，且候选元素一定是  m 。
### 2. 数学推导
设：
-  count  为计数器的值
-  m  为多数元素，出现次数为  k 
- 其他元素总出现次数为  n - k 
由于  k > n/2 ，则  n - k < n/2 。
**最坏情况**：每次  m  都与一个非  m  元素配对，抵消后剩余  k - (n - k) = 2k - n > 0  个  m 。
因此，最终候选元素一定是  m 。
## 四、算法步骤
1. **初始化**：
   - 候选元素 `candidate = nums[0]`
   - 计数器 `count = 1`
2. **遍历数组**：
   - 对于每个元素 `num`：
     - 如果 `num == candidate` → `count++`
     - 如果 `num != candidate` → `count--`
     - 如果 `count == 0` → 更新 `candidate = num`，`count = 1`
3. **返回**：`candidate`
## 五、代码实现
```java
public int majorityElement(int[] nums) {
    int candidate = nums[0];
    int count = 1;
    for (int i = 1; i < nums.length; i++) {
        if (candidate == nums[i]) {
            count++;
        } else {
            count--;
            if (count == 0) {
                candidate = nums[i];
                count = 1;
            }
        }
    }
    return candidate;
}
```
## 六、示例解析
### 示例1：`nums = [2,2,1,1,1,2,2]`
| 步骤 | 元素 | candidate | count | 操作 |
|------|------|-----------|-------|------|
| 1    | 2    | 2         | 1     | 初始 |
| 2    | 2    | 2         | 2     | 相同+1 |
| 3    | 1    | 2         | 1     | 不同-1 |
| 4    | 1    | 2         | 0     | 不同-1 |
| 5    | 1    | 1         | 1     | 更换候选 |
| 6    | 2    | 1         | 0     | 不同-1 |
| 7    | 2    | 2         | 1     | 更换候选 |
**结果**：`candidate = 2`
### 示例2：`nums = [3,2,3]`
| 步骤 | 元素 | candidate | count | 操作 |
|------|------|-----------|-------|------|
| 1    | 3    | 3         | 1     | 初始 |
| 2    | 2    | 3         | 0     | 不同-1 |
| 3    | 3    | 3         | 1     | 更换候选 |
**结果**：`candidate = 3`
## 七、适用场景
1. **多数元素问题**：寻找出现次数超过一半的元素
2. **数据流问题**：在线处理数据流中的多数元素
3. **内存受限场景**：空间复杂度要求严格的环境
## 八、变种与扩展
### 1. 寻找所有出现次数超过 n/3 的元素（LC229）
需要维护两个候选元素和两个计数器，最后需要验证候选元素是否满足条件。
### 2. 寻找出现次数最多的元素
摩尔投票算法不适用于此场景，需要使用哈希表统计频率。
## 九、面试常见问题
### 1. 摩尔投票算法的时间和空间复杂度？
**回答**：时间复杂度 O(n)，只遍历数组一次；空间复杂度 O(1)，无需额外空间。
### 2. 摩尔投票算法的适用条件？
**回答**：适用于存在多数元素的情况（出现次数超过一半）。如果不存在多数元素，算法会返回一个候选元素，但需要额外验证。
### 3. 如何验证摩尔投票算法的结果？
**回答**：遍历数组，统计候选元素的出现次数，检查是否超过数组长度的一半。
### 4. 摩尔投票算法的优缺点？
**回答**：
- **优点**：时间复杂度最优，空间复杂度为常数，适用于大规模数据。
- **缺点**：只适用于存在多数元素的情况，需要题目保证存在多数元素。
