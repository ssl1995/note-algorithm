# LC287_寻找重复数
## 题目描述
给定一个包含n + 1 个整数的数组nums ，其数字都在[1, n]范围内（包括 1 和 n），可知至少存在一个重复的整数。
假设nums只有一个重复的整数 ，返回这个重复的数 。
**要求**：
- 不能更改原数组（假设数组是只读的）
- 只能使用额外的O(1)空间
- 时间复杂度小于O(n²)
- 数组中只有一个重复的数字，但它可能不止重复出现一次
示例1：
```
输入：nums = [1,3,4,2,2]
输出：2
```
示例2：
```
输入：nums = [3,1,3,4,2]
输出：3
```
示例3：
```
输入：nums = [1,1]
输出：1
```
示例4：
```
输入：nums = [1,1,2]
输出：1
```

## 记忆口诀
寻找重复数，快慢指针法；数组变链表，环入口即答案
## 解法一：快慢指针（最优解）
### 思路
将数组视为一个链表，其中数组元素的值作为指向下一个节点的指针。由于数组长度为n+1，元素范围为[1,n]，必然存在环，而重复元素就是环的入口。
### 核心步骤
1. **初始化指针**：慢指针从nums[0]出发，快指针从nums[nums[0]]出发
2. **第一次相遇**：慢指针每次移动一步，快指针每次移动两步，直到相遇
3. **找环入口**：将快指针重置到起点，两者以相同速度移动，再次相遇的位置即为环的入口（重复元素）
### 图解过程
以nums = [1,3,4,2,2]为例：
1. **构建链表**：0→1→3→2→4→2（形成环：2→4→2）
2. **第一次相遇**：慢指针路径：0→1→3→2，快指针路径：0→1→3→2→4→2→4→2，在2处相遇
3. **找环入口**：快指针重置到0，慢指针在2，两者同时移动：0→1→3→2，2→4→2，在2处再次相遇，找到重复元素
### 代码示例
```java
public int findDuplicate(int[] nums) {
    if (nums == null || nums.length == 0) {
        return -1;
    }
    // 类似142—环形链表II：返回入环结点
    // 慢指针从起点出发，快指针从起点的下一个节点出发
    int slow = nums[0];
    int fast = nums[nums[0]];
    
    // 第一次相遇：慢指针走一步，快指针走两步
    while (slow != fast) {
        slow = nums[slow];
        fast = nums[nums[fast]];
    }
    
    // 第二次相遇：快指针重置到起点，两者以相同速度移动
    fast = 0;
    while (slow != fast) {
        slow = nums[slow];
        fast = nums[fast];
    }
    // 相遇点即为环的入口，也就是重复元素
    return slow;
}
```
### 复杂度分析
- 时间复杂度：O(n)，最多需要两次遍历数组
- 空间复杂度：O(1)，只使用常数级额外空间
### 优缺点
- 优点：时间复杂度最优，空间复杂度为常数，符合题目要求
- 缺点：逻辑较难理解，需要对环形链表有深入理解
### 测试用例
- 示例1：[1,3,4,2,2] → 期望输出：2
- 示例2：[3,1,3,4,2] → 期望输出：3
- 示例3：[1,1] → 期望输出：1
- 示例4：[1,1,2] → 期望输出：1
- 边界测试：[2,2,2,2,2] → 期望输出：2
### 边界情况分析
- 数组长度为2：直接返回唯一的元素
- 所有元素都相同：直接返回该元素
- 重复元素在数组开头：算法仍然有效
- 重复元素在数组末尾：算法仍然有效
### 易错点提醒
- 初始化指针时，快指针必须从nums[nums[0]]出发，而不是nums[0]
- 第一次相遇后，必须将快指针重置到0，而不是其他位置
- 第二次移动时，快慢指针必须以相同速度移动（每次一步）
### 原理解释
**为什么这种方法有效？**
1. **必然存在环**：数组长度为n+1，元素范围为[1,n]，根据鸽巢原理，至少有一个元素重复
2. **链表构建**：将数组下标视为节点，元素值视为指向下一个节点的指针，由于元素值在[1,n]之间，不会越界
3. **环的性质**：重复元素必然是环的入口，因为多个不同的下标会指向同一个值
4. **快慢指针**：利用快慢指针找到环的入口，这是环形链表II的经典解法
## 解法二：二分查找
### 思路
利用二分查找的思想，统计数组中小于等于mid的元素个数。如果个数大于mid，说明重复元素在[1,mid]之间；否则在[mid+1,n]之间。
### 核心步骤
1. **初始化边界**：left=1，right=n（n为数组长度-1）
2. **二分查找**：计算mid=(left+right)/2，统计小于等于mid的元素个数count
3. **调整边界**：如果count>mid，说明重复元素在左半部分，right=mid；否则在右半部分，left=mid+1
4. **返回结果**：当left=right时，找到重复元素
### 代码示例
```java
public int findDuplicate(int[] nums) {
    int n = nums.length - 1;
    int left = 1, right = n;
    while (left < right) {
        int mid = left + (right - left) / 2;
        int count = 0;
        // 统计小于等于mid的元素个数
        for (int num : nums) {
            if (num <= mid) {
                count++;
            }
        }
        // 如果个数大于mid，说明重复元素在左半部分
        if (count > mid) {
            right = mid;
        } else {
            left = mid + 1;
        }
    }
    return left;
}
```
### 复杂度分析
- 时间复杂度：O(n log n)，每次二分需要遍历整个数组
- 空间复杂度：O(1)，只使用常数级额外空间
### 优缺点
- 优点：思路直观，容易理解
- 缺点：时间复杂度较高，需要多次遍历数组
## 解法三：哈希表
### 思路
使用哈希表记录已经出现过的元素，遇到重复元素时直接返回。
### 代码示例
```java
public int findDuplicate(int[] nums) {
    Set<Integer> seen = new HashSet<>();
    for (int num : nums) {
        if (seen.contains(num)) {
            return num;
        }
        seen.add(num);
    }
    return -1;
}
```
### 复杂度分析
- 时间复杂度：O(n)，只需要遍历一次数组
- 空间复杂度：O(n)，需要额外的哈希表空间
### 优缺点
- 优点：思路简单，实现容易
- 缺点：使用了额外的O(n)空间，不符合题目要求
## 解法四：排序
### 思路
对数组进行排序，然后查找相邻的重复元素。
### 代码示例
```java
public int findDuplicate(int[] nums) {
    Arrays.sort(nums);
    for (int i = 1; i < nums.length; i++) {
        if (nums[i] == nums[i-1]) {
            return nums[i];
        }
    }
    return -1;
}
```
### 复杂度分析
- 时间复杂度：O(n log n)，排序的时间复杂度
- 空间复杂度：O(log n)，排序需要的栈空间
### 优缺点
- 优点：思路简单，实现容易
- 缺点：时间复杂度较高，且修改了原数组，不符合题目要求
