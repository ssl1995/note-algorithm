# LC72_编辑距离
## 题目描述
给你两个单词 word1 和 word2，返回将 word1 转换成 word2 所使用的最少操作数。
你可以对一个单词进行如下三种操作：
- 插入一个字符
- 删除一个字符
- 替换一个字符
示例1：
```
输入：word1 = "intention", word2 = "execution"
输出：5
解释：
intention → inention (删除 't')
inention → enention (将 'i' 替换为 'e')
enention → exention (将 'n' 替换为 'x')
exention → exection (将 'n' 替换为 'c')
exection → execution (插入 'u')
```
示例2：
```
输入：word1 = "horse", word2 = "ros"
输出：3
解释：
horse → rorse (将 'h' 替换为 'r')
rorse → rose (删除 'r')
rose → ros (删除 'e')
```
示例3：
```
输入：word1 = "", word2 = "a"
输出：1
解释：空字符串插入 'a'。
```
## 解法概览
| 解法 | 思路 | 核心公式 | 时间复杂度 | 空间复杂度 |
|------|------|----------|------------|------------|
| 二维DP | 动态规划 | `dp[i][j] = min(dp[i-1][j],dp[i][j-1],dp[i-1][j-1])+1` | O(m*n) | O(m*n) |
| 空间优化 | 滚动数组 | 只保留上一行 | O(m*n) | O(n) |
## 记忆口诀
编辑距离，DP来求；三操作取最小，相等则不变
## 解法一：二维动态规划
### 思路
定义 dp[i][j] 表示 word1 的前 i 个字符转换成 word2 的前 j 个字符所需的最少操作数。根据最后一个字符是否相等来推导状态转移。
### 核心公式
- 初始化：`dp[i][0] = i`（删除 i 次），`dp[0][j] = j`（插入 j 次）
- 状态转移：
  - `word1[i-1] == word2[j-1]`：`dp[i][j] = dp[i-1][j-1]`（不需要操作）
  - `word1[i-1] != word2[j-1]`：`dp[i][j] = min(dp[i-1][j], dp[i][j-1], dp[i-1][j-1]) + 1`（取三种操作的最小值+1）
### 图解过程
```
word1 = "horse", word2 = "ros"

dp表：
        r   o   s
    [0] [1] [2] [3]
h   [1]  1   2   3
    [2]  2   1   2
    [3]  3   2   2
    [4]  4   3   3
    [5]  5   4   3

填充过程：
- i=1,j=1: 'h'!='r' → min(1,1,0)+1=1 （替换操作）
- i=2,j=1: 'o'!='r' → min(2,2,1)+1=2 （替换操作）
- i=2,j=2: 'o'=='o' → dp[1][1]=1 （无需操作）
- 最终：dp[5][3]=3
```
### 代码示例
```java
public int minDistance(String word1, String word2) {
    int m = word1.length();
    int n = word2.length();
    // dp[i][j] = word1前i个字符转换为word2前j个字符的最少操作数
    int[][] dp = new int[m + 1][n + 1];

    // 初始化第一列：word2为空，需要删除i次
    for (int i = 1; i <= m; i++) {
        dp[i][0] = i;
    }

    // 初始化第一行：word1为空，需要插入j次
    for (int j = 1; j <= n; j++) {
        dp[0][j] = j;
    }

    // 填表
    for (int i = 1; i <= m; i++) {
        for (int j = 1; j <= n; j++) {
            if (word1.charAt(i - 1) == word2.charAt(j - 1)) {
                // 字符相等，无需操作
                dp[i][j] = dp[i - 1][j - 1];
            } else {
                // 字符不等，取三种操作的最小值+1
                // dp[i-1][j]：删除操作
                // dp[i][j-1]：插入操作
                // dp[i-1][j-1]：替换操作
                dp[i][j] = Math.min(
                    Math.min(dp[i - 1][j], dp[i][j - 1]),
                    dp[i - 1][j - 1]
                ) + 1;
            }
        }
    }

    return dp[m][n];
}
```
### 复杂度分析
- 时间复杂度：O(m*n)，m 和 n 分别为两个单词长度
- 空间复杂度：O(m*n)
### 优缺点
- 优点：思路清晰，状态定义明确
- 缺点：空间可以优化
### 测试用例
- 示例1：word1="intention", word2="execution" → 5
- 示例2：word1="horse", word2="ros" → 3
- 示例3：word1="", word2="a" → 1
- 边界：word1="", word2="" → 0
### 边界情况分析
- 空字符串：返回另一个字符串的长度
- 一个字符串是另一个的子串：差异长度
### 优化版本
```java
// 一维数组空间优化
public int minDistance(String word1, String word2) {
    if (word1.length() < word2.length()) {
        return minDistance(word2, word1);
    }
    int m = word1.length();
    int n = word2.length();
    int[] dp = new int[n + 1];

    for (int j = 1; j <= n; j++) {
        dp[j] = j;
    }

    for (int i = 1; i <= m; i++) {
        int prev = dp[0];  // 左上角的值
        dp[0] = i;         // 第一列更新
        for (int j = 1; j <= n; j++) {
            int temp = dp[j];
            if (word1.charAt(i - 1) == word2.charAt(j - 1)) {
                dp[j] = prev;
            } else {
                dp[j] = Math.min(Math.min(dp[j], dp[j - 1]), prev) + 1;
            }
            prev = temp;
        }
    }
    return dp[n];
}
```
### 解题技巧总结
- 技巧1：多一行一列处理边界情况（空字符串）
- 技巧2：字符相等时，直接继承左上角的值
- 技巧3：字符不等时，取三种操作的最小值+1
