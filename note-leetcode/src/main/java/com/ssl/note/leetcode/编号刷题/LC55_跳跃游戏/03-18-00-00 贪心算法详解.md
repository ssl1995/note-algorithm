# 03-18-00-00 贪心算法详解
## 什么是贪心算法
贪心算法是一种在每一步选择中都采取在当前状态下最好或最优（即最有利）的选择，从而希望导致结果是全局最优的算法。
## 核心思想
贪心算法的核心思想是**局部最优解累积得到全局最优解**。具体来说：
- 在每一步决策时，只考虑当前状态下的最优选择，不考虑未来可能的影响
- 每一步的选择都是局部最优的，且这些局部最优选择的集合构成全局最优解
## 基本要素
### 1. 贪心选择性质
**贪心选择性质**是指所求问题的整体最优解可以通过一系列局部最优的选择来达到。也就是说，在算法的每一步选择中，都做出在当前看来最好的选择，而不需要考虑子问题的解。

### 2. 最优子结构性质
**最优子结构性质**是指问题的最优解包含其子问题的最优解。当一个问题的最优解包含其子问题的最优解时，就称该问题具有最优子结构性质。
## 适用条件
贪心算法并非适用于所有问题，只有当问题同时满足以下条件时才能使用：
1. **贪心选择性质**：局部最优选择能导致全局最优解
2. **最优子结构性质**：问题的最优解包含其子问题的最优解
## 常见应用场景
1. **霍夫曼编码**：用于数据压缩，通过构建最优前缀码树
2. **活动选择问题**：在给定的活动集合中，选择最多的互不重叠的活动
3. **最小生成树**：如Kruskal算法和Prim算法
4. **单源最短路径**：如Dijkstra算法
5. **找零问题**：用最少的硬币数找零
6. **区间调度问题**：选择最多的互不重叠区间
7. **跳跃游戏**：判断是否可以到达终点
## 示例分析：跳跃游戏（LC55）
### 问题描述
给定一个非负整数数组 nums，你最初位于数组的第一个下标。数组中的每个元素代表你在该位置可以跳跃的最大长度。判断你是否能够到达最后一个下标。

### 贪心策略
- **局部最优**：在每一步，尽可能扩展能够到达的最远位置
- **全局最优**：通过不断扩展最远位置，判断是否能到达终点

### 核心公式
- maxReach：记录当前能够到达的最远位置
- 对于每个位置 i：
  1. 如果 i > maxReach，说明该位置不可达，返回false
  2. 更新 maxReach = max(maxReach, i + nums[i])
  3. 如果 maxReach >= nums.length - 1，返回true

### 代码示例（带详细注释）
```java
public boolean canJump(int[] nums) {
    if (nums == null || nums.length == 0) {
        return false;
    }
    
    int maxReach = 0; // 记录当前能够到达的最远位置
    
    for (int i = 0; i < nums.length; i++) {
        // 如果当前位置不可达，直接返回false
        if (i > maxReach) {
            return false;
        }
        
        // 更新最远可达位置
        maxReach = Math.max(maxReach, i + nums[i]);
        
        // 如果已经可以到达最后一个位置，直接返回true
        if (maxReach >= nums.length - 1) {
            return true;
        }
    }
    
    // 遍历完所有位置，检查是否可以到达最后一个位置
    return maxReach >= nums.length - 1;
}
```

### 复杂度分析
- 时间复杂度：O(n)，只需一次遍历数组
- 空间复杂度：O(1)，只需要常数级别的额外空间

### 示例验证
**输入：** [2,3,1,1,4]
- 初始：maxReach=0
- 位置0：0 <= 0 → maxReach=max(0, 0+2)=2
- 位置1：1 <= 2 → maxReach=max(2, 1+3)=4，4 >= 4（最后一个下标）→ 返回true

**输入：** [3,2,1,0,4]
- 初始：maxReach=0
- 位置0：0 <= 0 → maxReach=max(0, 0+3)=3
- 位置1：1 <= 3 → maxReach=max(3, 1+2)=3
- 位置2：2 <= 3 → maxReach=max(3, 2+1)=3
- 位置3：3 <= 3 → maxReach=max(3, 3+0)=3
- 位置4：4 > 3 → 返回false
## 贪心算法与其他算法的比较
### 与动态规划的区别
- **贪心算法**：每一步都做出局部最优选择，不回溯，时间复杂度较低
- **动态规划**：考虑所有可能的选择，通过记忆化或表格存储子问题的解，确保全局最优，时间复杂度较高

### 与回溯法的区别
- **贪心算法**：每一步选择不可撤销，直接进入下一步
- **回溯法**：尝试所有可能的选择，遇到不满足条件时回退，时间复杂度较高
## 优缺点
### 优点
- **时间复杂度低**：通常为O(n)或O(nlogn)，适用于大规模问题
- **实现简单**：代码逻辑清晰，易于理解和实现
- **空间复杂度低**：通常只需要常数级别的额外空间

### 缺点
- **并非总能得到最优解**：只适用于满足贪心选择性质的问题
- **需要证明正确性**：必须验证问题是否满足贪心选择性质和最优子结构性质
- **局部最优可能导致全局不优**：在某些问题中，局部最优选择可能无法累积为全局最优解
## 如何证明贪心算法的正确性
要证明贪心算法能解决某个问题，需要证明以下两点：

### 1. 贪心选择性质
证明每一步的局部最优选择能够导致全局最优解。通常采用反证法：假设存在一个全局最优解，通过调整该解，使其包含贪心选择，且调整后的解仍然是最优的。

### 2. 最优子结构性质
证明问题的最优解包含其子问题的最优解。通常采用数学归纳法：假设对于规模较小的子问题，贪心算法能得到最优解，那么对于规模较大的问题，贪心算法也能得到最优解。
## 总结
贪心算法是一种高效的算法设计策略，通过每一步的局部最优选择来尝试获得全局最优解。它适用于具有贪心选择性质和最优子结构性质的问题，如霍夫曼编码、活动选择、最小生成树、跳跃游戏等。

虽然贪心算法并非适用于所有问题，但其简洁性和高效性使其成为算法设计中的重要工具。在实际应用中，需要根据问题的特点判断是否适用贪心算法，并通过数学证明或反例验证其正确性。对于不满足贪心选择性质的问题，可能需要使用动态规划、回溯法等其他算法策略。

掌握贪心算法的思想和应用，对于解决各类算法问题和系统设计问题都有重要意义。
